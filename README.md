## How to Reproduce the Results

`dev.txt` and `test.txt` can be regenerated by running either **`BestModel/model.ipynb`** or **`4_model/model.ipynb`** from top to bottom. Both notebooks implement the same pipeline; `BestModel/model.ipynb` points to the saved checkpoints in `BestModel/` directly, while `4_model/model.ipynb` is the full experimentation notebook used during development.

**Default mode (fast — loads saved weights):**

Both notebooks default to `TRAIN_MODE = False`, which skips retraining and loads the saved checkpoints from `BestModel/roberta-base/` and `BestModel/distilbert/`. Running all cells will:

1. Load and preprocess the data.
2. Load the saved RoBERTa and DistilBERT checkpoints.
3. Run the 2-D ensemble weight and threshold grid search on the validation set.
4. Evaluate on the official dev set (held-out test) and report F1.
5. Write `dev.txt` (2,094 lines) and `test.txt` (3,832 lines) to the repo root.

**To retrain from scratch:**

Set `TRAIN_MODE = True` in the configuration cell. This trains both backbones with 3 seeds each (seeds 42, 7, 123), selects the best checkpoint per model, and then runs the same evaluation and prediction steps above. Training requires a GPU and takes approximately 4–6 hours on a Tesla T4.

```bash
pip install -r requirements.txt
```

## Model Approach

### Problem and Baseline

Binary classification with a severe class imbalance: only ~9.5% of paragraphs (794 out of 8,375 training examples) are labelled PCL (`orig_label >= 2`). The official RoBERTa-base baseline achieves F1 = 0.48 on the dev set.

### Proposed Approach: Dual-Backbone Ensemble

The submitted system is a weighted probability ensemble of `roberta-base` and `distilbert-base-uncased`, both fine-tuned with the following techniques:

**1. Input Enrichment via Metadata Prepending**  
Each paragraph is prepended with its keyword and country using special entity tokens:
```
<e>keyword</e> <e>country</e> {paragraph text}
```

**2. Weighted Random Sampler (WRS)**  
Uses square-root-scaled inverse-frequency weights (softer than full inverse-frequency) to produce balanced mini-batches. Ambiguous positives (annotator agreement score of 2/4) receive an additional 0.5× down-weight to reduce noise from borderline labels.

**3. Grouped Layer-wise Learning Rate Decay (LLRD)**  
Transformer layers are split into three groups; lower layers receive a smaller learning rate and upper layers a larger one (decay factor λ = 1.6). This preserves general pre-trained representations while allowing task-specific adaptation near the classifier head.

**4. Cosine Annealing Scheduler with Linear Warmup**  
Linear warmup over the first 10% of training steps followed by cosine decay, promoting stable early training and smooth convergence.

**5. Mid-Training Evaluation, Early Stopping, and Multi-Seed Selection**  
Validation F1 is evaluated every 50 gradient steps; the best model state is captured whenever a new best is reached. Early stopping triggers after 15 consecutive evaluation checks without improvement. Each backbone is trained with three random seeds (42, 7, 123) and the checkpoint with the highest validation F1 is kept.

**6. Ensemble Weight and Threshold Search**  
After selecting the best checkpoint per backbone, a 2-D grid search over the RoBERTa weight `w ∈ [0, 1]` and decision threshold `t ∈ [0.05, 0.95]` is performed on the internal validation set to maximise F1. An automatic fallback selects the solo RoBERTa model if the ensemble does not improve on validation F1. The optimal weights found were 0.50/0.50 with threshold 0.50.

### Final Results

| System | Val F1 | Dev Set F1 | Threshold |
|---|---|---|---|
| Solo RoBERTa (best config, seed=7) | 0.5795 | 0.5955 | 0.75 |
| Solo DistilBERT (best config, seed=7) | 0.5818 | 0.5463 | 0.50 |
| **Ensemble (w=0.50/0.50)** | **0.5882** | **0.6069** | **0.50** |

The ensemble outperforms both individual models and was selected as the final submission.
